Building a Database

Role: Database Administrator • Tools: Draw.io, Oracle APEX •
Time: 2 weeks

Purpose: To gain experience in database development by creating a simple recipe storage database that uses relationsl database techniques.

Identifying and Defining

The first step was to identify the entities and attributes that would be necessary. Defining the Primary and Foreign Keys came next.

Entities

Ingredient
Cookware
Recipe

Composite Tables

Ingred-Recipe
Cookware-Recipe

Primary and Foreign Keys

Ingredient_ID
Cookware_ID
Recipe_ID



Entity Relationship Diagram



Next I mapped out how entities would be logically related. The focus was on understanding what data would need to interact with other data, while keeping data integrity in mind. I started by creating the tables that were most vital to the data: Ingredient, Recipe, and Cookware.

From here, I worked to understand how these tables would interact with each other. The Recipe table would have to be able to access information about ingredients and what cookware would be used for multiple recipes, so in order to prevent duplicates in the data, I created composite tables, that allow for many recipes to access many ingredients as well as many cookware items.


Queries

After solidfying the relationships between tables using the Entity Relationship Diagram, I moved on to creating the database in Oracle APEX using SQL queries. I started by making each table, which included specifying attributes, data types, and constraints like the primary and foreign keys.



The next steps were to insert data into the tables and verify everything with test queries.



Outcome

The outcome of my exploration in database creation was a simple database that takes ingredients, quantities, cookware, and directions, then compiles everything into complete recipes.
